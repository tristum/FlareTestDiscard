utilizes to satisfy the log review and analysis component of our security best practices. Each day, System Security Analysts will use to analyze event log data, track and escalate incidents, and send notifications that pertain to the systems that generate these problematic logs. utilizes standards such as PCI-DSS to provide a prescriptive framework to generate reports. These reports address specific requirements within the standards.

The service is designed to meet the following compliance mandates:

  • Review of daily logs
  • Analyze event log data for potential security incidents
  • Send notification for incidents that warrant further investigation
  • Create an audit trail for auditors and regulators

Amazon CloudTrail logs are not included in Daily service; however, CloudTrail reports are available for you to use within the .

Service level commitment

Every 24 hours, a System Security Analyst will review the logs from the previous day and escalate any associated incidents in accordance with your escalation process.

Daily review actions

If a security or compliance incident is identified, a System Security Analyst will contact you with the details of the alert and offer any recommendations. The analyst will maintain a full LogReview audit trail which is accessible through the Alert Logic User Interface for future audit reporting. The analysis will include only logs provided by active and configured hosts.

Audit trail

For each daily LogReview task, a case is created and stored in the Log Manager section of the Alert Logic User Interface. You will be able to view the following content for any date covered within the LogReview scope:

  • Report results

  • Analyst comments

  • Analyst actions taken (if any)

Reports can be used to prove to an auditor that monitoring is taking place. This audit trail is maintained for a minimum of 12 rolling months.

Command prompt host mode installation

Use this installation when there is one dedicated virtual server. With this installation, displays log data collected from the installed host agent under one associated source in the . You cannot use the agent in host mode to capture an image of the virtual machine.

To install a the agent in host mode via the command prompt:

  1. Copy the MSI file to the target machine.
  2. Use the following command to install the agent in host agent mode:

    msiexec /i [path to MSI file] prov_only=host prov_key=[unique registration key]

    To understand the above command: 

    • /i installs the agent normally.
    • prov_only=[host] sets the agent as a role type, provisions the agent, and runs the agent to start collection.
    • [prov_key=[unique registration key] is your Unique Registration Key.

     

     

    msiexec /i c:\downloads\-1.0.33.msi prov_only=host prov_key=

    • /i installs the agent normally
    • prov_only=[host] sets the agent as a host type, provisions the agent, and runs the agent to start collection.
    • [prov_key=[unique registration key] is your unique registration key
    • Command prompt example: msiexec /i c:\downloads\-1.0.33.msi prov_only=host prov_key=
  3. Verify that the agent has registered with the . To do so, in the left navigation window, under Collection, click Sources, and then search for your agent.

Agent registration may take several minutes.

  1. (Optional) Enable flat-file collection.
    Note To enable flat-file collection in , see Create a log source.

Command prompt autoscaling mode installation

Use this installation when a number of virtual servers perform the same function. With this installation, displays log data collected from the agent under one associated source in the . Use the agent in autoscaling mode for load balancing. When you install the agent in autoscaling mode, you must install the agent without starting the agent. The image capture works only if the agent is installed and has never started autoscaling. When the agent starts autoscaling for the first time, it registers itself with . If images are captured using the agent already autoscaling, there will be multiple agents with the same registration details that will report to .

To install the agent in autoscaling mode via the command prompt:

  1. Copy the MSI file to the target machine.
  2. Use the following command to install the agent in autoscaling mode:

    msiexec /i [path to MSI file] install_only=1 prov_only=role prov_key=[unique registration key]

    To understand the above command: 

    • /i installs the agent normally.
    • prov_only=[role]sets the agent as a host type, provisions the agent, and runs the agent to start collection.
    • [prov_key=[unique registration key] is your unique registration key.
    • /i installs the agent normally
    • [install_only=1] installs the agent without running it
    • prov_only=[role] sets the agent as a host type, provisions the agent, and runs the agent to start collection.
    • [prov_key=[unique registration key] is your unique registration key

     

    msiexec /i c:\downloads\-1.0.33.msi install_only=1 prov_only=role prov_key=

    • Command line example: msiexec /i c:\downloads\-1.0.33.msi install_only=1 prov_only=role prov_key=
  3. (Optional) If you have set up a NAT or virtual appliance and you want to use it as a single point of egress, enter the corresponding host name or IP address or port via the following command prompt parameters:
    • sensor_host=[host] indicates the IP address where the agent should forward logs.
    • sensor_port=[port] indicates the TCP port where the agent should connect.

    For other command prompts parameters, see Optional Windows command-prompt installation parameters.

Install the Agent for Linux

Install the agent on RPM-based distributions

  1. Copy RPM package to the target machine.
  2. If you run SELinux, you must first run the following command:  semanage port -a -t syslogd_port_t -p tcp 1514

If the semanage command is not present in your system, you can install the policycoreutils-python package to obtain the semanage command. recommends that you consult with your system administrator to verify.

  1. Run the following command:
    rpm -i -<version>*.rpm
  2. Run one of the following commands based on your preference:
    • To configure the agent in host mode, run:
      /etc/init.d/ provision --key <UNIQUEREGISTRATIONKEY> --inst-type host
    • To configure the agent in auto scaling mode, run:
      /etc/init.d/ provision --key <UNIQUEREGISTRATIONKEY> --inst-type role

For more information about agent mode types, see About the Alert Logic agent host mode and auto scaling mode

  1. Run the following command: 
    /etc/init.d/ start

Do not run this command if you want to capture the image of a virtual machine.

  1. If you use an rsyslog daemon, add the following line to rsyslog.conf:
    *.* @@127.0.0.1:1514;RSYSLOG_FileFormat

This configuration will direct your local syslog to the agent on TCP port 1514.

  1. If you use a syslog-ng daemon, add the following lines to syslog-ng.conf:
    • destination d_alertlogic {tcp("localhost" port(1514));};
    • log { source(s_src); destination(d_alertlogic); };

This configuration will direct your local syslog to the agent on TCP port 1514.

  1. Restart the syslog daemon.
  2. Verify that the agent has registered with the . To do so, in the left navigation window, under Collection, click Sources, and then search for the agent.

The agent registration may take several minutes.

  1. (Optional) Capture an image.

Note You can capture an image of the virtual machine that contains the installed agent. See Capture an image.

6 Configure collection policies

Create flat-file policy

  1. At the top of the , from the drop-down menu, select Log Manager.
  2. In the left navigation window, under Policies, click Flat File.
  3. In the table of collection policies, in the Actions column, click the gear icon (), and select New FFC Policy.
  4. In Flat-File Policy Name, type a descriptive policy.
  5. In the Source File Path field, type the path

    6. To use an agent for collection, specify the local file system path to the log files. Otherwise, specify the network share path to the log files.

  6. In the File Name or Pattern field, type the file name or date pattern of the flat-file log messages.

    7. htaccess.* is a file name with a pattern. The * represents the time stamp of the flat-file log. Indicate the specific time format for this flat-file. accepts a variety of date formats.

  7. From File Name Rotation Scheme, select a file name rotation scheme. The format must match the format of your flat-file log messages.

    The default, Auto-Detect identifies many rotation schemes. recommends you specify the rotation scheme format of your flat-file log messages. If you are unsure of the format or do not find the specific format from the drop-down menu, select Auto-Detect.

  8. In Multiline Handling, select a multiline handling option:
    • If your flat-file log messages contain single lines, keep the selection: File contains single line log messages.
    • If your flat-file log messages contain multiple lines, select File contains log messages with multiple lines. Also, select and enter a configuration.
      • If the length of your log messages is consistent:
        • Keep the selection: Each log message spans a fixed number of lines, and then in Number of lines, type the number of lines.
      • If the length of your log messages is not consistent:
        • Select Each log message follows a known pattern, select the appropriate Pattern application, and then type the Pattern that takes place in the log message. Next, select Regular expression to use a Perl Compatible Regular Expression (PCRE).
  9. In Timestamp Rule, select a timestamp rule option:
    • To use the timestamp from the collector, keep the selection: Set message time as collect time.
    • To use an existing timestamp, select Parse times from messages using a pre-defined timestamp format. Next, select a format from Format a date string.
    • To use a custom timestamp, select Parse times from messages using a custom timestamp format. Next, enter a format for the date string in the expanded configuration area.
      • In the Format of date string field, type a format for the date string, and follow the on screen instructions, and select Check format to validate your specified pattern.
  10. In Host Credential, select or create a credential:

If you use the for log collection, do not select or create host credentials.

To enable log collection with a remote collector installed in a physical appliance used to collect flat-files or Windows event log messages or a remote collector installed in the cloud used to collect S3 log messages, you must provide host credentials. An expired host credential will stop log collection. Update any host credential changes in the collection policy.

  • To use an existing credential, keep the default selection: Use an existing credential. Next, from Existing Credential, select a credential.
  • To create a new credential, select Create a new credential. Next, enter new credentials.
    • In the corresponding configuration fields, type a Credential Name, Host Username, and Host Password. Next, in the Retype Password field, retype the host password.
  1. In Collection Schedule, select or create a collection schedule:
    • To select a collection schedule, keep the default selection: Use an existing schedule. Next, from Existing schedule, select a schedule.
    • To create a new collection schedule, select Create a new schedule. Next, enter and select the schedule options.
